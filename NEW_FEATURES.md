## 1. 代码架构重构

### 🎯 目标
将单文件 4434 行的 App.jsx 拆分成模块化结构，提高可维护性和可扩展性。

### 📁 新的文件结构

```

├── firebase/              # Firebase 配置和数据库操作
│   ├── config.js          # Firebase 初始化
│   ├── firestore.js       # Firestore CRUD 操作
│   ├── familyService.js   # 家庭组服务
│   └── messaging.js       # FCM 推送消息
├── contexts/              # React Context
│   └── AuthContext.jsx    # 认证状态管理
├── services/              # 业务逻辑服务
│   ├── migrationService.js    # 数据迁移服务
│   └── notificationService.js # 通知服务
├── components/            # UI 组件
│   ├── ProgressSection.jsx    # 5×5×5 进度面板
│   ├── AnalyticsPanel.jsx     # 数据分析面板
│   ├── RecipeModal.jsx        # 食谱选择弹窗
│   ├── DailySummaryModal.jsx  # 每日汇总弹窗
│   ├── MealSelectorModal.jsx  # 餐次选择弹窗
│   ├── ShoppingListModal.jsx  # 购物清单弹窗
│   ├── FoodSettingsModal.jsx  # 食材设置弹窗
│   ├── auth/                  # 认证相关组件
│   │   ├── LoginModal.jsx
│   │   └── RegisterModal.jsx
│   ├── family/                # 家庭共享组件
│   │   └── FamilyGroup.jsx
│   └── notifications/         # 通知组件
│       └── NotificationSettings.jsx
├── data/                  # 数据常量
│   ├── defenseSystems.js      # 五大防御系统
│   ├── nutritionDB.js         # 营养成分数据库
│   ├── foodHealthDB.js        # 食材健康数据库
│   ├── recipesDB.js           # 食谱数据库
│   ├── dietPatterns.js        # 饮食模式定义
```

## 2. Firebase 云同步

### 🔥 功能
- 所有数据自动同步到 Firebase Firestore
- 支持多设备实时同步
- 离线缓存支持

### 📦 数据结构

```
users/{userId}
├── healthProfile        # 健康档案
├── nutritionGoals       # 营养目标
├── healthGoal          # 健康目标
├── dietaryPreferences  # 饮食偏好
├── favoriteRecipes     # 收藏食谱
├── cookingHistory      # 烹饪历史
└── mealHistory         # 用餐历史

fridges/{fridgeId}
├── ownerId             # 所有者 ID
├── type                # personal/family
└── foods/              # 食材子集合
    └── {foodId}
        ├── name, quantity, grams
        ├── expiryDate, createdAt
        ├── healthData
        └── addedBy

familyGroups/{groupId}
├── name                # 家庭名称
├── inviteCode          # 邀请码
├── memberIds[]         # 成员列表
├── adminId             # 管理员
└── familyFridgeId      # 共享冰箱 ID
```

### 🔄 同步机制
- **实时监听**：使用 Firestore `onSnapshot` 实时同步
- **乐观更新**：本地先更新，再同步到云端
- **离线支持**：Firestore 内置离线缓存

### 📊 功能

### 📝 记录方法

2. 点击「编辑档案」

### 📉 趋势图显示
- **需要至少 2 次记录**才会显示
- 蓝色折线 = 体重趋势
- 绿色折线 = 体脂率趋势
- 数据点显示具体数值
- X 轴显示日期

## 10. 食谱推荐优化

### 🎯 推荐优先级

| 优先级 | 因素 | 权重 |
|--------|------|------|
| 1 | 临期食材 | 最高 |
| 2 | 饮食模式匹配 | 高 |
| 3 | 大满贯食材 | 中 |
| 4 | 健康目标调整 | 中 |
| 5 | 热量预算 | 低 |

### 📊 排序逻辑

1. **排除已做过的菜**
2. **按紧急程度排序**
   - 🔴 红色（3 天内到期）= 500 分
   - 🟠 橙色（3-7 天到期）= 300 分
   - 🟢 绿色（7 天以上）= 0 分
3. **按共同食材分组**
   - 包含相同食材的菜集中显示
4. **同组内按分数排序**

### 🍽️ 食谱数量
- **总计**：200+ 道菜谱
- **地中海饮食**：15 道
- **高蛋白饮食**：8 道
- **低碳水饮食**：6 道
- **素食**：6 道
- **生酮饮食**：5 道
- **抗炎饮食**：5 道

## 11. 每日自动清零

### 🔄 清零内容

| 数据 | 清零时间 | 说明 |
|------|---------|------|
| 用餐记录 (meals) | 每天 0 点 | 自动重置为空数组 |
| 热量计算 | 每天 0 点 | 基于 meals，自动清零 |
| 营养分析 | 每天 0 点 | 基于 meals，自动清零 |
| 5×5×5 进度 | 不清零 | 基于冰箱食材，持续累积 |

### ⚙️ 实现方式

```javascript
// meals 初始化时检查日期
const saved = localStorage.getItem("fridge_meals");
const today = new Date().toDateString();
if (saved) {
  const parsed = JSON.parse(saved);
  // 只保留今天的数据
  return parsed.date === today ? parsed.meals : [];
}
return [];
```
